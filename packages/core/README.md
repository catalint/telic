# @telic/core

A recorded intent + memory layer for frontends — record-first user-intent lifecycle with
in-page queryable session memory.

```bash
npm install @telic/core
# or
bun add @telic/core
```

The full pitch, worked examples with scrutiny notes, the "Isn't this just X?" comparison, and
the roadmap live in the [repository README](https://github.com/catalint/telic#readme).

The design is documented in full alongside this package:

- [SPEC.md](./SPEC.md) — **what** (the normative, clause-numbered contract)
- [DESIGN.md](./DESIGN.md) — **why** (boundaries, risks, the initiative boundary)
- [PATTERNS.md](./PATTERNS.md) — **how** (patterns P1–P10, anti-patterns AP1–AP8)
- [APPROACHES.md](./APPROACHES.md) — **which** (per-axis decision guide)
- [DECISIONS.md](./DECISIONS.md) — **when / what changed** (append-only decision log)
